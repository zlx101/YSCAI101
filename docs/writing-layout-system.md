# YSC Warm Editorial 写作排版规范

本规范用于成云杉 YSC 网站的长文排版。它参考 `isjiamu/gzh-design-skill` 的主题化组件思路，但不直接复制公众号组件源码。公众号排版关注“粘贴不掉格式”，本网站关注“长期统一、响应式阅读、个人站气质”。

## 参考结论

`gzh-design-skill` 的 6 套主题里，最适合本站的是三个方向的交集：

- **石墨极简**：灰阶承重、细线分隔、克制强调，适合正文阅读。
- **留白禅意**：衬线标题、弱容器、大留白，适合个人思考与深度随笔。
- **橄榄手记**：编者按、对比摘要、流程/清单组件，适合 AI 实践、产品复盘和案例拆解。

不作为本站主风格：

- **摸鱼绿**：适合公众号教程和工具盘点，信息密度高，但偏工具号。
- **摸鱼票据**：票据隐喻鲜明，适合创意评测，不适合全站统一。
- **红白色系**：观点力度强，但正红过重，和本站暖纸色、陶土色系统不完全一致。

## 设计原则

1. **内容优先**：正文阅读比装饰重要。任何组件都必须帮助读者理解结构、判断重点或行动步骤。
2. **灰阶承重，陶土点睛**：正文由深炭、灰褐承重；陶土色只用于锚点、标签、重点线和按钮。
3. **少量稳定组件**：长期只保留 8 类文章组件，避免每篇文章现场发明样式。
4. **卡片要有语义**：卡片只用于导读、提醒、对比、步骤、资产、CTA；普通段落不套卡片。
5. **不用虚线框强调**：虚线只用于“待补素材”占位。强调用左竖条、小标签、细线或浅底。
6. **每段少量标记**：重点短语可用下划线或加粗，但每段最多 1-2 处，不把整段画花。
7. **移动端先成立**：目录、步骤、对比、资产网格都必须能自然落成单列。

## 统一组件

| 内容语义 | 网站组件类 | 使用场景 |
| --- | --- | --- |
| 文章封面摘要 | `.post-cover` | 长文开头，给出一句核心判断 |
| 开头导读 / 编者按 | `.post-summary-card` | 第一屏后，压缩文章主旨 |
| 目录导航 | `.article-index` | 4 个以上章节时使用 |
| 引用 / 金句 | `blockquote` / `.post-pullquote` | 强观点、章节转场 |
| 重要提醒 | `.post-callout` / `.post-note` | 风险、边界、核心提醒 |
| 步骤 / 流程 | `.post-step-grid` | 教程、方法、闭环 |
| 清单 | `.post-checklist` / `.post-number-list` | 行动建议、分类要点 |
| 对比 / 资产 | `.post-compare` / `.post-assets` | 两种做法、长期沉淀 |

## 文章类型配方

### AI 实践 / 工具教程

结构：封面摘要 → 核心判断 → 步骤网格 → 清单 → 代码块 / 提示块 → 结尾提醒。

可用组件：`.post-step-grid`、`.post-checklist`、`.post-number-list`、`pre code`、`.post-callout`。

### 观点 / 深度分析

结构：封面摘要 → 导读卡 → 目录 → 正文分节 → 引用金句 → 对比卡 → 结尾收束。

可用组件：`.post-summary-card`、`.article-index`、`blockquote`、`.post-compare`、`.post-pullquote`。

### 产品复盘 / 案例拆解

结构：封面摘要 → 背景 → 问题 → 做法 → 结果 → 复盘资产。

可用组件：`.post-step-grid`、`.post-assets`、`.post-note`、`.post-number-list`。

### 个人介绍 / 随笔

结构：短导读 → 低密度正文 → 少量居中金句或左竖条提示 → 温和结尾。

可用组件：`blockquote`、`.post-pullquote`、`.post-pill-grid`，少用重卡片。

## 颜色与字体

- 背景：沿用网站暖纸色背景，不为文章单独铺大面积深色。
- 正文：深炭 / 暖黑，保证长时间阅读。
- 辅助文字：灰褐色。
- 主强调：陶土色 `#bb5d37` 附近。
- 辅助强调：墨绿 `#315f49` 附近，用于 check、可信感、流程稳定感。
- 标题：文章标题可用粗黑体；章节标题优先使用衬线字体或暖黑粗体，保持个人站气质。

## 写作落版规则

1. 文章首屏必须有：返回链接、文章编号/栏目、标题、日期/阅读时间/作者、标签。
2. 1000 字以上长文必须有导读卡；章节多于 4 个建议加目录。
3. 每个 `h2` 只承载一个判断，不写空泛标题。
4. 卡片之间保持 24-32px 间距，避免连环卡片造成公众号味太重。
5. 单篇文章的强强调组件不超过 5 处。
6. 结尾必须回到行动、判断或长期资产，不用泛泛鸡汤收尾。

## 与公众号 Skill 的边界

保留：

- 主题索引 / 组件库 / 配方表的思路。
- 细线、左竖条、小标签、目录、导读卡、对比卡、步骤卡。
- “不要随机混搭组件”的原则。

不保留：

- 全内联样式、`span leaf`、公众号平台校验。
- 每段强制 1-3 个下划线关键词。
- 票据、硬阴影、强红色、过高信息密度。
- 为粘贴公众号而牺牲语义 HTML 的限制。

以后新增文章，优先使用本规范里的组件和配方；确实需要新组件时，先问它解决了什么阅读问题，再加入 CSS。
